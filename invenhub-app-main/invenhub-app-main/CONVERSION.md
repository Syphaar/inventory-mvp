- ============================================================================
- KEY COMPONENTS & THEIR PURPOSE
- ============================================================================
-
- 1.  src/main.tsx
- - Application entry point
- - Initializes React Router DOM with BrowserRouter
- - Sets up React Query QueryClientProvider
- - Renders App component into #root element
-
- 2.  src/App.tsx
- - Root route configuration
- - Defines public routes (/login, /register)
- - Defines protected routes (wrapped in ProtectedRoute)
- - Wraps entire app with AuthProvider
- - Global providers (Toaster for notifications)
-
- 3.  src/context/AuthContext.tsx
- - Global authentication state management
- - Provides user, token, login, register, logout functions
- - Persists token to localStorage
- - Automatically restores session on page reload
- - useAuth() hook for accessing auth state in components
-
- 4.  src/api/apiClient.ts
- - Centralized Axios instance
- - Request interceptor: Attaches JWT token to all requests
- - Response interceptor: Handles 401 (token refresh/redirect)
- - Global error handling
- - All API calls use this client
-
- 5.  src/components/ProtectedRoute.tsx
- - Route guard component
- - Redirects unauthenticated users to /login
- - Shows loading state while checking authentication
- - Wraps protected routes in App.tsx
-
- 6.  src/components/layout/Layout.tsx
- - Main layout wrapper for authenticated pages
- - Provides sidebar navigation and responsive design
- - User profile section with logout
- - Mobile hamburger menu

- ============================================================================
- AUTHENTICATION FLOW
- ============================================================================
-
- 1.  User navigates to /login
- 2.  Enters email and password
- 3.  Calls AuthContext.login()
- 4.  Sends POST /api/auth/login to backend
- 5.  Backend returns { user, accessToken }
- 6.  AuthContext stores token in localStorage and state
- 7.  apiClient interceptor attaches token to subsequent requests
- 8.  ProtectedRoute checks isAuthenticated and allows navigation to /dashboard
- 9.  On logout, token cleared and user redirected to /login
-
- Token Persistence:
- - Token saved in localStorage on login
- - Token restored from localStorage on app start (useEffect in AuthContext)
- - API client automatically includes token in Authorization header
- - 401 responses trigger logout and redirect to /login
-
- ============================================================================
- STATE MANAGEMENT STRATEGY
- ============================================================================
-
- Authentication State (Global):
- - Managed by AuthContext
- - Includes: user, token, isAuthenticated, isLoading, error
- - Persisted to localStorage
- - Accessed via useAuth() hook
-
- Page/Component State (Local):
- - Managed with useState() for form inputs, UI state, etc.
- - Products, sales, purchases loaded via useEffect
- - Data refetching on demand or at intervals
-
- Caching:
- - React Query (already installed) ready for caching API responses
- - Can be implemented for GET endpoints to reduce API calls
- - Currently loads fresh data on each navigation

- import { productApi } from '@/api';
-
- function MyComponent() {
- const [products, setProducts] = useState([]);
-

-     const loadProducts = async () => {
-       const data = await productApi.getAll();
-       setProducts(data);

-     loadProducts();

- }

-
- import { useAuth } from '@/hooks';
-
- function Header() {

- }
